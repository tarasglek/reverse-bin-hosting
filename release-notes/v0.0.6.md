# Reverse Bin v0.0.6

## Highlights

- Static HTML applications work again with the managed Unix sockets introduced in v0.0.5.
- Package CI now installs the built Debian package and exercises both static and executable applications through the real Caddy handler.
- Releases must include categorized, human-authored notes before CI will publish them.

## Breaking changes

None. Existing applications and configuration files remain compatible.

## Full list of changes

- Bundle caddy-reverse-bin v0.3.2, which creates static-app socket directories before launch and removes them during cleanup.
- Update Caddy and vulnerable transitive Go dependencies used by the packaged proxy.
- Add installed-package integration coverage for static Unix-socket routing, executable routing, packaged modules, detector version, and service health.
- Fetch the latest caddy-reverse-bin release directly from GitHub when refreshing runtime pins, avoiding stale Go proxy results.
- Require release notes with Highlights, Breaking changes, and Full list of changes sections.
